# Leanbox

A fitness-focused e-commerce platform (Philippines, PHP) selling vegetarian meals, high-protein
packages, supplements, healthy snacks, wellness products, and recurring meal-prep subscriptions.
Three roles: **admin**, **customer**, **rider**.

> Project context, stack decisions, and business rules live in [`CLAUDE.md`](CLAUDE.md).
> Planning docs (PRD, ERD, architecture, etc.) live in [`docs/`](docs/).

## Monorepo layout

```
leanbox/
├─ backend/    # Laravel API (PHP 8.2+, Sanctum, PostgreSQL via Supabase)
├─ frontend/   # Next.js (App Router) + Tailwind
├─ docs/       # PRD, ERD, system architecture, tech stack, UI/UX, roadmap
└─ CLAUDE.md   # source-of-truth project context
```

## Tech stack

- **Backend:** Laravel 12, Eloquent, Sanctum. PostgreSQL + image storage via **Supabase**.
  Cache/queue/session on the **database** driver (no Redis). Payments: **COD only** for now.
- **Frontend:** Next.js + Tailwind (+ shadcn/ui, Framer Motion, TanStack Query, Zustand).

## Getting started

### Backend (`backend/`)
```bash
cp .env.example .env          # fill in Supabase DB + storage credentials
composer install
php artisan key:generate
php artisan migrate --seed     # builds schema + demo data
php artisan serve              # http://localhost:8000
```
Requires PHP 8.2+ with the `pdo_pgsql`/`pgsql` extensions enabled.
Demo logins after seeding (password `password`): `admin@leanbox.test`, `customer@leanbox.test`, `rider@leanbox.test`.

### Frontend (`frontend/`)
```bash
npm install
npm run dev                    # http://localhost:3000
```

## API conventions

- **Versioned base path:** `/api/v1`. Admin routes under `/api/v1/admin/*`, rider under `/api/v1/rider/*`.
- **Response envelope:** every API response uses `{ data, meta, error }`.
  - Success → `data` set, `error` null, `meta` carries pagination/extra info.
  - Error → `data`/`meta` null, `error` = `{ code, message, details? }`.
  - Build responses with `App\Support\ApiResponse::success()` / `::error()`.
- **Errors** are normalized centrally in `bootstrap/app.php` (validation → 422, auth → 401,
  forbidden → 403, not found → 404, etc.).
- **Auth:** Laravel Sanctum bearer tokens.
- **OpenAPI / docs:** auto-generated by Scramble, served at `/docs/api` (UI) and `/docs/api.json`
  (spec). The frontend generates its TypeScript types from this spec — keep request/response
  types accurate (Form Requests + API Resources).

## Contributing conventions

- **Branches:** work on **`develop`**; **`main`** is reserved for PRs (`develop → main`).
  Don't commit directly to `main`.
- **Commits:** Conventional Commits style — `type(scope): summary`
  (`feat`, `fix`, `chore`, `docs`, `refactor`, `test`). Example:
  `feat(catalog): add product search endpoint`.
- **Pull requests** must use this structure (PR = documentation):
  - **Summary** — concise overview
  - **WHAT** — all changes made
  - **HOW** — implementation approach, key files, migrations, dependencies
  - **WHY** — problem solved, improvements, trade-offs
  - **Acceptance Criteria** — reviewer checklist
  - **Notes** — limitations, follow-ups, deployment considerations
- **CI:** `backend-ci.yml` runs Pint + migrations + Pest on every push/PR; keep it green.
- **Never commit secrets.** `.env` is gitignored; use `.env.example` as the template.

## Testing

- Backend: `php artisan test` (Pest). Frontend: `npm run test` (Vitest). E2E: Playwright.
